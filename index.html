n8n 是什麼？

一個開源、視覺化的工作流程自動化工具

簡單來說，n8n 就像一盒數位樂高積木。每一個「節點 (Node)」就是一塊不同功能的積木 (例如：收信、寫入表格)，我們可以輕易地將這些積木堆疊、串接起來，打造出專屬於我們自己的自動化城堡。

搞懂這 4 個詞，你就學會了 n8n

Workflow (工作流程)

整個自動化的腳本。

Node (節點)

腳本中的一個步驟。

Trigger (觸發器)

腳本的開頭。

Execute (執行)

運行單一節點或整個流程來測試結果。

Workflow (工作流程)

這是您整個自動化流程的「畫布」。所有的節點都會在這裡被串連起來，形成一個完整的自動化腳本，決定了資料如何從頭到尾流動。

Node (節點)

節點是組成 Workflow 的基本「積木」。每一個節點都代表一個特定的應用或操作，例如「從 Google Sheets 讀取資料」或「發送一則 Slack 訊息」。

Trigger (觸發器)

觸發器是 Workflow 的「起點」，也是一種特殊的節點。它決定了這個自動化腳本何時、或如何開始執行。例如：「每當收到新的 Webhook 請求時」或「每天早上九點」。

Execute (執行)

這是 n8n 中最重要的測試與除錯功能。每當您完成一個節點的設定，都可以點擊「Execute Step」按鈕來立即運行它，並在右側的 Output 視窗查看結果，確保資料如您預期地被處理。

我們的自動化瑞士刀

Webhook

n8n的「耳朵」，接收即時訊號的萬用起點。

Google Chat

n8n的「嘴巴」，發送即時通知到內部群組。

Schedule (排程)

n8n的「鬧鐘」，可以定時自動啟動流程。

HTTP Request

n8n的「雙手」，主動抓取或傳送資料到任何API。

Webhook

n8n的「耳朵」，接收來自外部服務的即時訊號，是啟動許多自動化流程的萬用起點。任何支援 Webhook 的應用程式都可以透過它將資料傳送給 n8n。

Google Chat

n8n的「嘴巴」，可以將處理好的資料或任何訊息，即時發送到指定的內部群組或個人。是完成自動化後，最常用的通知工具。

Schedule (排程)

n8n的「鬧鐘」。當您的流程需要定時自動啟動，而不是被動等待觸發時，Schedule 節點就是您的最佳選擇。可以設定每分鐘、每小時、每天或任何自訂的時間來執行。

HTTP Request

這是 n8n 中最萬用的節點之一。它讓您的自動化流程可以主動向任何網站或 API 發出請求來抓取資料 (GET)，或是將資料傳送過去 (POST)。我們今天實作的範例就會用到它！

案例分享：新人教官 2.0 技術核心

痛點 (1.0 版本):

舊版教官使用傳統的「關鍵字比對」搜尋，不僅速度較慢，而且只要提問的用詞和文件標題不同，就常常找不到答案。

價值 (2.0 版本):

新版導入 **RAG (Retrieval-Augmented Generation) 向量檢索生成** 技術。它不再是死板地比對文字，而是能真正「理解」問題的語意，先從知識庫中找出最相關的幾份文件，再交給 AI 總結出最精準的答案。

案例分享：MIXXIN 魔法師

痛點：

行銷或設計同仁需要快速產出高品質的視覺素材，但手動設計耗時，而直接使用 AI 繪圖又需要複雜的 Prompt 技巧，門檻較高。

價值：

提供一個整合了 100+ 優質 Prompt 範例的簡易前端介面，任何人都能一鍵生成風格獨特的圖片。n8n 在其中扮演了穩定、安全的後端，負責接收前端請求並呼叫 Gemini 圖片生成模型。
