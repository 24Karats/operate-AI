n8n 是什麼？

一個開源、視覺化的工作流程自動化工具

簡單來說，n8n 就像一盒數位樂高積木。每一個「節點 (Node)」就是一塊不同功能的積木 (例如：收信、寫入表格)，我們可以輕易地將這些積木堆疊、串接起來，打造出專屬於我們自己的自動化城堡。

搞懂這 4 個詞，你就學會了 n8n

Workflow (工作流程)

整個自動化的腳本。

Node (節點)

腳本中的一個步驟。

Trigger (觸發器)

腳本的開頭。

Credentials (憑證)

操作其他APP的鑰匙。

Workflow (工作流程)

這是您整個自動化流程的「畫布」。所有的節點都會在這裡被串連起來，形成一個完整的自動化腳本，決定了資料如何從頭到尾流動。

Node (節點)

節點是組成 Workflow 的基本「積木」。每一個節點都代表一個特定的應用或操作，例如「從 Google Sheets 讀取資料」或「發送一則 Slack 訊息」。

[ 此處放入點擊 "+" 新增節點的介面截圖 ]

Trigger (觸發器)

觸發器是 Workflow 的「起點」，也是一種特殊的節點。它決定了這個自動化腳本何時、或如何開始執行。例如：「每當收到新的 Webhook 請求時」或「每天早上九點」。

[ 此處放入觸發器節點 (如 Webhook) 的設定畫面截圖 ]

Credentials (憑證)

這是授權 n8n 操作您其他應用程式帳號的「鑰匙」。例如，當您要讓 n8n 讀取您的 Google Sheets 時，就需要提供 Google 帳號的憑證，讓 n8n 獲得存取權限。

我們的自動化瑞士刀

Webhook

n8n的「耳朵」，接收即時訊號的萬用起點。

Google Chat

n8n的「嘴巴」，發送即時通知到內部群組。

Cron (排程)

n8n的「鬧鐘」，可以定時自動啟動流程。

IF (條件判斷)

n8n的「大腦」，讓流程學會思考與判斷。

Webhook

n8n的「耳朵」，接收來自外部服務的即時訊號，是啟動許多自動化流程的萬用起點。任何支援 Webhook 的應用程式都可以透過它將資料傳送給 n8n。

[ 此處放入 Webhook 節點的 URL 截圖 ]

Google Chat

n8n的「嘴巴」，可以將處理好的資料或任何訊息，即時發送到指定的內部群組或個人。是完成自動化後，最常用的通知工具。

[ 此處放入 Google Chat 節點的訊息設定截圖 ]

Cron (排程)

n8n的「鬧鐘」。當您的流程需要定時自動啟動，而不是被動等待觸發時，Cron 節點就是您的最佳選擇。可以設定每分鐘、每小時、每天或任何自訂的時間來執行。

[ 此處放入 Cron 節點的時間設定截圖 ]

IF (條件判斷)

n8n的「大腦」，它讓您的自動化流程學會思考與判斷。您可以根據收到的資料內容，設定不同的條件，讓流程走向不同的分支，執行不同的後續動作。
